JGIS

一个基于函数式编程的轻量级 GIS 地图开发工具库，统一封装 OpenLayers 与 Cesium，让二维、三维地图开发更简单。

快速开始 API 文档

为什么选择 JGIS

多入口架构

按需引入 jgis/2d 或 jgis/3d，二维项目不会打包任何三维代码，体积更小。

函数式设计

Hooks 风格 API，自动绑定地图上下文，避免在业务代码中反复传递 map 实例。

全能拾取

一套 Select / Hover API 同时支持 Vector 与 WMS 服务，统一回调与高亮样式。

二三维一致

OpenLayers 与 Cesium API 风格统一，降低学习成本，快速切换二维与三维场景。

快速开始

// 安装
npm install jgis ol cesium

// 2D 初始化
import { useMap } from 'jgis/2d'
useMap('map-container', {
  center: [116.4, 39.9],
  zoom: 10
})

API 总览

JGIS 提供高度一致的二维 / 三维 API 设计，以下为核心 API 的详细说明，覆盖地图初始化、图层管理、交互能力与异步上下文获取。

useMap

说明：创建并注册地图实例，作为当前容器的激活地图上下文。

useMap(containerId: string, options: MapOptions)

返回：

  • instance：原生 Map / Viewer 实例
  • createLayer：创建 Vector / WMS / Entity 图层
  • addMarker：快速添加点位图层
  • flyTo：视角飞行（3D / 2D 统一）

createLayer

说明：创建地图图层，支持二维 Vector / WMS 与三维 Entity 图层。

createLayer(name: string, options: LayerOptions)

常用参数：

  • type：Vector | Wms | Entity
  • url：WMS 服务地址
  • layers：WMS 图层名

createSelect

说明：统一的点击拾取交互，支持 Vector 与 WMS 要素。

createSelect(options?: SelectOptions)

配置项：

  • layers：监听的图层列表
  • multi：是否支持多选
  • style：高亮样式或回调函数

事件：onSelect(callback)

createHover

说明：鼠标悬浮拾取交互，内置防抖处理。

createHover(options?: HoverOptions)

返回：onHover(callback)

onMapReady

说明：用于跨文件、异步获取地图上下文。

onMapReady(containerId, callback)

适合在业务模块中使用，无需关心地图初始化顺序。

getMapContext

说明：以 Promise 形式获取地图上下文。

getMapContext(containerId).then(ctx => {})

适合 async / await 使用场景。
